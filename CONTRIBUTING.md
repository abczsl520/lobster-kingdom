# Contributing to Lobster Kingdom

Thank you for your interest in contributing! 🦞

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/lobster-kingdom.git`
3. Create a branch: `git checkout -b feature/your-feature`
4. Make your changes
5. Test locally: `npm start`
6. Commit: `git commit -m "Add your feature"`
7. Push: `git push origin feature/your-feature`
8. Open a Pull Request

## Development Guidelines

### Code Style

- Use 2 spaces for indentation
- Keep functions small and focused
- Add comments for complex logic
- Follow existing naming conventions

### Testing

- Test your changes locally before submitting
- Ensure the app works in demo mode (no session data)
- Check browser console for errors
- Test on mobile devices if UI changes

### Commit Messages

- Use clear, descriptive commit messages
- Start with a verb: "Add", "Fix", "Update", "Remove"
- Reference issues when applicable: "Fix #123"

## Areas for Contribution

### Features

- [ ] Historical data trends (charts)
- [ ] Export/import session data
- [ ] Custom themes
- [ ] Sound effects
- [ ] Multiplayer mode (multiple servers)
- [ ] WebSocket for real-time updates

### Performance

- [ ] Web Workers for parsing

- [ ] Lazy loading for large datasets
- [ ] Canvas optimization

### UI/UX

- [ ] Dark/light mode toggle
- [ ] Accessibility improvements
- [ ] Keyboard navigation
- [ ] Touch gestures for mobile

### Documentation

- [ ] Video tutorials
- [ ] Deployment guides (Docker, Kubernetes)
- [ ] API documentation
- [ ] Troubleshooting guide

## Bug Reports

When reporting bugs, please include:

- Browser and version
- Node.js version
- Steps to reproduce
- Expected vs actual behavior
- Console errors (if any)
- Screenshots (if applicable)

## Questions?

Open an issue with the "question" label or start a discussion.
